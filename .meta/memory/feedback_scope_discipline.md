---
name: scope-discipline
description: Do not conflate revision of existing content with creation of new content. Editing post #1 means editing post #1 — not writing a different post.
type: feedback
---

When asked to revise/edit/amend existing content, the output must be a revised version of THAT content — same topic, same core material, better structure. Not a new piece that references the original.

**Why:** The user asked to revise post #001 (documentation infrastructure credentials). I wrote a completely different post about the contribution engine and Testament formalization, then called it a "revision." These are two separate content pieces. The user caught the scope drift immediately.

**How to apply:** Before writing, ask: "Am I revising the existing content or creating new content?" If revising, the subject matter stays the same. The structure, argument, and presentation improve. The topic does not change.
