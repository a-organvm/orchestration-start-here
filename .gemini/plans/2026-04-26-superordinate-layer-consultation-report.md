## 1. "What Is": The Current State

The ecosystem currently consists of a highly developed execution layer (the "Hokage" per-domain executors) and a nascent meta-organizational layer (portfolio synthesis).

## 2. "What's Missing": The Vacua and Gaps

Using the scientific framework where unknowns are *vacua* to be discovered rather than *gaps* to be penalized, the following elements are currently missing:

1. **The Superordinate Layer Itself:** The theoretical "four operators" (selfish-altruistic loop, magnetic membrane, portfolio, and reflexive) exist on paper but have no codebase implementation. The layer is currently a vacuum.
2. **N-way Portfolio Combination Criteria:** The synthesis layer is open, but lacks the specific recombination rules detailing *how* atoms combine.
3. **Escalation Contract:** There is no hard operational boundary defining what constitutes a "systemic" versus a "local" conflict. 
4. **Reflexive Loop Anchor:** The system lacks an external anchor (human validation) to break the infinite regress of a self-evaluating governance model.
5. **Cold Verification (Stranger Test):** `SKILL.md` and the foundational substrate remain self-attested; they have not been verified by a fresh, cold-reading agent.
6. **Repo Discovery:** 20 of the 37 ecosystem repositories have only been mapped via name-inference and require a structural audit. Note: Several "missing" repos (`mirror-mirror`, `a-i-council`, `gamified-coach-interface`) exist in the `bench/` archive but are not active.

## 3. "What Should Be": The Vision

The superordinate layer must act as the **body that contains the organs**, rather than just another organ. 
- **Role:** It should function as an *observatory* that detects the seams and vacua radiated by the Hokage executors, managing cross-domain pattern detection and portfolio-level resource arbitration.
- **Evolution:** It should begin as a loosely coupled *tool* (advisory), earning its authority by demonstrating accurate pattern detection, before eventually evolving into a constraining *system*.
- **Stream Orchestration:** Stream Τ (tooling) and Stream Ω (corpus processing) should operate seamlessly in parallel, as they address different tiers of the ecosystem (integration vs. atomic mapping).
