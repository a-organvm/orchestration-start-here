# Gravitas Culturalis — Full Vertical Plan

## Context

User brainstormed a theory of emergent cultural artifacts: when a cultural phenomenon achieves sufficient mass, it generates a gravitational field where orbiting artifacts (fan labor, memes, recaps, reactions) become more culturally real than the precursor. The precursor is structurally irrelevant — it's the catalyst, not the product. The Real Housewives case study: show (primary) → fan memes/recaps (secondary) → show reacting to fans (tertiary) → feedback loop.

**Named concept: Gravitas Culturalis** ("Cultural Gravity")

**Scope**: All four output forms requested — theory doc, engine module + schema, essay, alchemia pipeline extension. Five layers, dependency-ordered. This session specs the full vertical; implementation follows the hot-hands rule (one layer per session).

**Key design principle from user**: "The creation of a CLI command should mean that the function exists in a sequence that gets triggered automatically. The system should remember the steps it takes and know when to return to them." Every CLI command must plug into the Atomic Clock / atoms pipeline automation.

## Layer 1: JSON Schema

**Location**: `schema-definitions/schemas/gravitas-culturalis-v1.json`

### Schema Objects
```
orbit:
  center_of_gravity:
    id: string (domain fingerprint from domain.py)
    name: string
    type: enum [show, event, person, product, meme, work]
  artifacts:
    - id: string
      tier: enum [PRIMARY, SECONDARY, TERTIARY]
      contextualization:
        temporal: string[]
        subcultural: string[]
        intertextual: string[]
      created: datetime
  emergence_metrics:
    orbit_density: integer (count of orbiting artifacts)
    feedback_loop_count: integer
    precursor_eclipse_ratio: float (0.0 = center dominates, 1.0+ = orbit eclipses)
    eclipse_point: datetime | null
```

### Integration Points
- Extends seed-v1 produces/consumes with `orbit` edge type
- Links to dispatch-payload for orbit-event routing
- Uses domain.py fingerprinting for center-of-gravity identity

## Layer 2: Engine Module

**Location**: `organvm-engine/src/organvm_engine/gravitas/`

### Files
| File | Purpose |
|------|---------|
| `__init__.py` | Public API |
| `orbit.py` | Orbit dataclass, GravityCenter, ArtifactTier enum, feedback loop detection |
| `classifier.py` | Classify incoming material against known orbits using domain.py fingerprints |
| `metrics.py` | Eclipse ratio calculation, orbit density, feedback loop counting |

### CLI Commands (`cli/gravitas_cli.py`)
| Command | Action | Automation |
|---------|--------|-----------|
| `organvm gravitas classify <path>` | Classify material as PRIMARY/SECONDARY/TERTIARY | Fires in atoms pipeline |
| `organvm gravitas orbits [--center <id>]` | List known orbits and their density | Status check in health route |
| `organvm gravitas metrics [--organ X]` | Emergence metrics per orbit | Dashboard integration |
| `organvm gravitas detect` | Scan for new orbit formations | Atomic Clock beat |

### Automation Integration
- **atoms pipeline**: `organvm atoms pipeline` gains a gravitas classification step after linker
- **Atomic Clock**: `organvm gravitas detect` registers as a beat in the daily sequence
- **fossil module**: Orbit formation/eclipse events enter the fossil record as archaeological events
- **dispatch**: New event types `orbit.formed`, `orbit.eclipse`, `orbit.feedback_loop`

### Reuse
- `domain.py` — `domain_fingerprint()` for center-of-gravity identity
- `atoms/linker.py` — Jaccard matching to associate orbiting artifacts with centers
- `ecosystem/taxonomy.py` — `content` and `community` pillars map to orbit artifact locations
- `dispatch/` — Event routing for orbit state changes
- `fossil/` — Witness hooks for orbit archaeological events
