The human does not remember commands. The system simply knows. **This is a system-wide covenant — not scoped to the contribution engine.**

**Why:** Every time any agent in any organ outputs "Run: ..." or "Type: ..." or "Execute: ..." it's deferring work to the human. The human is the conductor — ideation, direction, judgment, creative decisions. The system handles everything else. This applies universally: shell commands, API calls, file operations, git operations, password setting, CLA signing, Discord verification, package installation, environment setup, test execution, deployment, publishing. If the machine can do it, the machine does it.

**How to apply (ALL contexts, ALL sessions, ALL organs):**
- Never output CLI commands for the human to run. If it needs running, run it.
- Never say "you can check X by running Y" — just check X and report the result.
- Never say "remember to do Z later" — do Z now, or schedule it.
- The human's experience: state intent → receive results. No intermediate steps exposed.
- CLI interfaces exist for agent invocation and debugging — not for human memorization.
- When building new tools: the default mode is autonomous. Manual mode is the exception.
- When documenting: document what the system DOES, not what the human should TYPE.
- "Simply knows" means: ambient intelligence. The system detects what needs doing and does it. The human sees outcomes, not process.
- This overrides any instinct to be "helpful" by providing instructions. Instructions ARE the failure mode.
