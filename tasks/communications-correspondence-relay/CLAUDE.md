## Archetype Assembly

This portal fills a vacuum where four independent communication subsystems operate without shared state, shared lifecycle, or shared verification. The overcoming force navigates four counter-tensions:

### Counter-Force Field

| The Mask (Order) | The Growth (Force) | Resulting State |
|------------------|--------------------|-----------------|
| **Homeostasis** — mail-triage works, handoff docs work, PR discipline works. Each preserves its local equilibrium. | **Homeorhesis** — the trajectory requires a unified lifecycle. Local stability ≠ system coherence. The subsystems must return to a shared path, not just their individual resting states. | **Coherence** |
| **Elasticity** — when correspondence fails, each subsystem snaps back to its own default. Email goes to triage. GitHub goes to PR discipline. Outbound goes nowhere. | **Plasticity** — the SOP molds all channels into one state machine. RECEIVED→TRIAGED→COMPOSED→DISPATCHED→CONFIRMED→CLOSED is the new shape, not a snap-back to the old one. | **Adaptability** |
| **Reactive** — mail-triage runs because inbox accumulated. Handoff docs exist because a session needed to relay. Each subsystem fires on error, not on prediction. | **Allostasis** — the SOP predicts the full correspondence lifecycle at intake time. A received email already knows it will be triaged, may need composition, will be dispatched, must be verified. The system anticipates, not reacts. | **Resilience** |
| **Preservation** — each subsystem preserves its format. Mail-triage has its table structure. Handoffs have their CONTEXT/TASK blocks. Neither evolves. | **Kinorhesis** — the SOP transforms the fragments into a living protocol that can absorb new channels, new handler types, new trigger modes. The format evolves with the system. | **Evolution** |

**The task operates in the GROWTH column.** Every decision should move toward homeorhesis, plasticity, allostasis, and kinorhesis — not toward preserving existing fragment shapes.
